# 認証フロー詳細設計 (2026/03/16)

## 概要
Googleログイン（実装済み）に加え、メールアドレスベースの新規登録・サインインフローを構築する。
メール送信には Resend を利用し、セキュリティ確保のために署名付きURL（マジックリンク）を使用する。

## 認証フロー

### 1. サインイン・開始画面
- ユーザーがメールアドレスを入力し、送信ボタンを押下。
- サーバー側で以下を実行：
    - `FERNET_KEY` を使用して、`{ email, timestamp }` を含むトークンを暗号化・署名。
    - マジックリンク URL を生成： `/auth/setup-password?token={encrypted_token}`
    - Resend API を使用して、生成した URL をユーザーのメールアドレスに送信。

### 2. パスワード設定画面 (Setup / Reset Password)
- マジックリンクから遷移。
- サーバー側でトークンの復号・署名検証・有効期限チェックを実行。
- 有効な場合、以下の入力を促す：
    - **メールアドレス**: (固定表示)
    - **表示名**: (既存ユーザーの場合は現在の名前を表示、新規の場合は email の @ より前)
    - **新しいパスワード**: (初期値: email)
- 保存ボタン押下で DB (`users`, `user_passwords`) を登録、または更新（上書き）。
- **パスワード失念時もこのフローを再利用することで、安全に再設定が可能。**
- 更新完了後、自動的にログインセッションを開始。

### 3. 既存の Google ログイン
- 現在の実装を維持。
- 初回ログイン時にパスワードが未設定の場合、同様のパスワード設定画面へ誘導することを検討。

## 技術仕様

### 環境変数
- `RESEND_API_KEY`: Resend の API キー
- `FERNET_KEY`: トークンの暗号化・復号用キー

### データベース (既存スキーマの活用)
- `users`: `id`, `email`, `display_name`
- `user_passwords`: `user_id`, `password_hash`

### セキュリティ
- トークンには有効期限（例：30分）を持たせ、期限切れの場合は再入力を促す。
- 復号に `FERNET_KEY` を用いることで、DB に一時的なトークンを保存せずにステートレスな検証を可能にする。
