# Markdownガイド

`qps5` では、プランの説明やコメント欄で **Markdown（マークダウン）** 形式による装飾が可能です。テキストを構造化して、読みやすい分析結果を作成しましょう。

## 1. 基本的な書式

| 装飾 | 書き方 | プレビュー |
| :--- | :--- | :--- |
| **太字** | `**テキスト**` | **テキスト** |
| *斜体* | `*テキスト*` | *テキスト* |
| ~~打ち消し~~ | `~~テキスト~~` | ~~テキスト~~ |
| `コード` | \` `SELECT` \` | `SELECT` |

## 2. 見出し
行頭に `#` を付けることで見出しになります。

```markdown
# 見出し1（タイトル級）
## 見出し2（セクション級）
### 見出し3（小見出し）
```

## 3. リスト
行頭に `*` または `-` を入れると箇条書きになります。数字の場合は `1.` を使います。

```markdown
* 原因の特定
* 修正案の提示
  * インデックス追加
  * クエリ書き換え
```

## 4. コードブロック（重要）
SQL文や実行プランの断片を載せる際に便利です。言語名（sql）を指定するとハイライトされます。

```sql
-- インデックス追加案
CREATE INDEX IX_TargetColumn ON MyTable (TargetColumn);
```



## 5. 引用
他人のコメントや、ドキュメントの一部を引用する際に使用します。

```markdown
> このプランでは Index Scan が発生しており、
> パフォーマンス低下の原因となっています。
```

## 6. テーブル
データの比較や、統計情報のサマリーを載せるのに最適です。

```markdown
| 項目 | 修正前 | 修正後 |
| :--- | :--- | :--- |
| CPU時間 | 1200ms | 150ms |
| 論理読込数 | 45,000 | 320 |
```


## 7. リンク
`[表示テキスト](URL)` の形式でリンクを貼ることができます。
例: `[MS公式ドキュメント](https://learn.microsoft.com/...)`

---

## 💡 チューニング議論のコツ
* **数値は正確に**: CPU時間や I/O 数は、テーブル機能を使って「Before / After」を明記すると説得力が増します。
* **コードブロックを活用**: 提案する SQL 修正案は、必ずコードブロックで囲むことでコピー＆ペーストしやすくなります。
* **チェックリスト**: `* [ ]` を使うと、TODOリスト（確認事項）を作成できます。
  * [ ] インデックスの断片化を確認した
  * [ ] 統計情報を更新した
